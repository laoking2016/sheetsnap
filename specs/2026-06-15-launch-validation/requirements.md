# 阶段 5：上线 & 验证 — 需求规格

> 对应 roadmap 阶段 5。目标：验证产品价值，获取 50 个真实用户，再决定是否投入收费系统。

---

## 范围

### 包含

| 模块 | 内容 | 优先级 |
|------|------|--------|
| Landing Page | 首页改为正式的营销页面，清晰传达价值主张 | P0 |
| Vercel 部署 | 推送到 GitHub → 自动部署到 Vercel（Preview + Production） | P0 |
| 免费使用 | 所有用户无需登录即可免费上传，不设次数限制 | P0 |
| 使用量埋点 | 记录上传次数、解析成功率、错误类型 | P1 |
| 用户行为分析 | 注册率、留存率、回访率追踪 | P1 |
| 解析失败收集 | 用户上传失败时可选择发送文件供改进 | P1 |
| Reddit 推广 | 在 r/procurement 发帖获取初始流量 | P2 (人工) |
| Upwork 用户研究 | 付费招募采购经理试用并录制反馈 | P2 (人工) |

### 不包含

- ❌ 付费墙 / Stripe 集成（阶段 4）
- ❌ 多语言支持
- ❌ SEO / 博客推广

---

## 关键决策记录

| 决策 | 选项 | 选定 | 理由 |
|------|------|------|------|
| 部署平台 | Vercel / Railway / Fly | **Vercel** | 与 README.md 一致，免费额度足够 |
| 域名 | Vercel 默认 / 自定义 | **自定义域名** | 先上默认域名，阶段 6 再绑定自定义域名 |
| 分析工具 | Vercel Analytics / Plausible / PostHog | **Vercel Analytics** | 零配置，与 Vercel 原生集成 |
| 失败文件收集 | 邮件发送 / Supabase Storage | **Supabase Storage** | 已有 Supabase 项目，配置简单 |
| 用户反馈 | 内联表单 / 邮件 / 第三方 | **内联表单 → feedback 表** | 已有的 `feedback` 表直接用 |
| 注册门槛 | 强制注册 / 不注册可用 | **不注册可用** | 降门槛，先用产品再决定是否注册 |

---

## 验证指标定义

| 指标 | 定义 | 数据来源 | 目标值 |
|------|------|---------|--------|
| 注册用户数 | 完成 Google OAuth 或 Magic Link 登录的用户数 | Supabase `users` 表 | ≥ 50 |
| 解析成功率 | 成功返回结果的请求 / 总上传请求 | API 日志 / 埋点 | ≥ 85% |
| 7 天回访率 | 第 1 天访问的用户，第 7 天再次访问的比例 | 埋点 | ≥ 20% |
| 上传转化率 | 上传至少 1 个文件的访客 / 总访客 | 埋点 | ≥ 40% |
