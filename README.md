# 产品名称：Sheetsnap
# 1. 目标用户画像（MVP 阶段）
- 核心用户：中小型海外B2B企业的采购经理 / 采购助理。
- 公司规模：10-200人，年采购额50万-500万美元。
- 典型状态：每周至少收到10份来自中国/东南亚供应商的Excel或PDF报价单。用户自己的- ERP系统老旧或根本无自动导入功能，必须手动逐行录入。
- 最痛场景：报价单里的产品编码/描述/单位/价格格式乱七八糟，经常录错，月底对账时才发现。
# 2. 核心场景（一句话）
采购经理收到一封含报价单附件的邮件 → 上传到SheetSnap → 几秒内得到一份干净的CSV表格（列：内部产品编码、产品名称、规格、单价、最小起订量、备注）→ 复制粘贴到自己的ERP或采购模板中。
# 3. MVP 功能范围（只做这些，其他一律不做）
模块 | 功能 | 优先级 | 说明
--- | --- | --- | ---
文件上传 | 支持 .xlsx, .xls, .csv, .pdf | P0 | 只处理表格类文件；PDF必须是表格式（非扫描图片版暂时不支持）
文件上传 | 支持拖拽上传 | P1 | 提升体验，但不是MVP必须
智能解析 | 自动识别表格区域（跳过标题/公司LOGO等无关行） | P0 | 使用现成AI解析API（如LlamaParse / Unstructured.io）
智能解析 | 自动映射到标准列：产品名称、规格、单价、最小起订量、货币 | P0 | 最多5个标准列，其他列暂时丢弃或作为“其他信息”保留
智能解析 | 自动标准化单位（如把“pcs”→“个”，“kg”→“公斤”），如果无法识别则保留原文并高亮 | P1 | 简化版可用规则映射，不做复杂NLP
结果输出 | 生成CSV文件下载 | P0 | UTF-8编码，兼容Excel打开
结果预览 | 在线表格预览前10行 | P1 | 方便用户快速检查
错误处理 | 解析失败时给出明确提示（如“未检测到表格，请确保文件中包含行列结构的报价数据”） | P0 | 减少客服咨询
计费与账户 | Google登录 / 邮箱注册 | P1 | MVP先用试用+手动发放额度，不强制注册
计费与账户 | 每月免费20次解析，超出后按次付费 | P1 | 初期通过Stripe收款
## 明确不做（MVP）：
- 不做多行业模板学习（用户教系统规则）
- 不做ERP/QuickBooks等第三方系统自动同步
- 不做PDF扫描图片OCR（即手写/拍照的报价单不支持）
- 不做多语言界面（只做英语）
- 不做团队协作/权限管理
# 4. 用户操作流程（每一步）
```text
1. 用户打开 sheetsnap.com
2. 首页大按钮：“Upload Quotation File”
3. 用户拖拽或点击上传文件（单个文件 ≤ 10MB）
4. 系统显示“解析中...”（5秒内完成）
5. 解析成功后，显示预览表格（标准5列）
   - 若某个单元格无法解析，显示⚠️图标，鼠标悬浮显示原文
6. 用户检查后点击“Download CSV”
7. 系统记录本次解析次数，若未登录则提示“输入邮箱接收下载链接”（同时记录邮箱）
8. 下载完成，用户打开CSV复制到自己的采购系统
```
## 异常流程：
- 上传非表格类文件 → 提示“请上传Excel或PDF格式的报价单”
- 解析失败 → 显示“未能识别出表格，你可以手动复制粘贴，或联系我们帮你优化”
- 达到免费限额 → 弹出支付页面（Stripe）
# 5. 技术选型建议（低成本快速验证）
部分 | 推荐技术 | 理由
--- | --- | ---
前端 | Next.js (React) + Tailwind CSS | 快速开发，部署在Vercel免费
后端 | 直接使用Next.js API Routes | 无需单独服务器
文件存储 | 上传后暂存内存，处理完即删；如需持久化用AWS S3（可选） | 保护用户数据隐私
核心解析 | LlamaParse (by LlamaIndex) 或 Unstructured.io | 专门处理非结构表格，准确率高，有免费额度；不推荐自研OCR
表格数据清洗 | Python Pandas (通过API调用) 或 纯JavaScript | Pandas对表格清洗更强大，可部署为独立微服务
认证 | NextAuth.js + Google OAuth | 简单
支付 | Stripe Checkout | 海外标准
部署 | Vercel + Supabase (PostgreSQL) | 免费额度足够MVP

**关键点：不要自己训练模型**，直接调用现成的文档解析API，你的价值在于“针对B2B报价单场景的列映射和单位归一化”。
# 6. 定价策略（验证付费意愿）
- 免费层：每月20次解析，无需绑定信用卡（仅需邮箱）
- 付费层（MVP阶段只做一个档位）：
  - **$19/月** → 200次/月，超出按$0.1/次
  - 或按量：$9.9 买100次（永不失效）
- 企业定制（MVP不做）: $99/月，包含API对接

**定价逻辑**：采购经理时薪约$25-40，每张报价单手动录入需5-15分钟，相当于每次浪费$2-8。如果产品收$0.1-0.2/次，价值感很强。

# 7. 验证指标（MVP上线后2周）
- 核心指标：
  - 注册用户数（至少50个）
  - 解析成功率（≥85%）
  - 付费转化率（从免费到付费 ≥ 3%）
  - 用户留存：7天内回访率 ≥ 20%
- 定性反馈：
  - 主动要求支持更多列（如“交期”“包装”）
  - 主动询问API
  - 抱怨不支持图片PDF（说明需求真实）
- 渠道验证：
  - 在Reddit r/procurement 发帖“我做了个免费工具帮你解析报价单”，看自然点击和私信量
  - 在Upwork上找5个兼职采购，送$10礼品卡让他们试用并录制1分钟反馈视频
# 8. 风险与备选
风险 | 应对
--- | ---
核心解析API费用超支 | 初期严格限制文件大小，对免费用户添加水印解析；若用户量大，换用开源方案（如surya-ocr + table transformer）自己部署
用户要求支持扫描图片PDF | 第二版本加上（需增加OCR引擎），MVP明确写不支持，但收集需求
大厂快速复制 | 这类工具竞争壁垒不在技术，而在数据积累和行业理解。先跑通一个垂直品类（比如电子元器件报价单），做深做透
用户不愿意上传敏感报价单 | 强调“处理完立即删除”，不出售数据，可签署数据保护承诺；提供自托管选项（后续）
